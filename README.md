# ToDo CLI App

Консольное приложение для управления списком задач (ToDo List), написанное на языке Dart. Реализует базовые CRUD-операции: создание, просмотр, завершение и удаление задач. Приложение демонстрирует работу с коллекциями, системой Null Safety, именованными параметрами, а также подключение сторонних пакетов и основы асинхронности.

 Что изучили в ходе лабораторной работы
Основы синтаксиса Dart: статическая типизация, вывод типов (var), работа с неизменяемыми переменными (final и const).
Система Sound Null Safety: безопасная работа с nullable-типами (String?), предотвращение ошибок null reference на этапе компиляции.
ООП-особенности языка: сокращённая запись конструкторов, именованные конструкторы, именованные и опциональные параметры функций, переопределение методов (@override, toString()).
Управление зависимостями: структура pubspec.yaml, установка пакетов через dart pub add, использование официального реестра pub.dev.
Асинхронность: концепция Future, механизм event loop, неблокирующее ожидание через async/await.
 Ответы на контрольные вопросы
Чем final отличается от const в Dart?
final определяет переменную, значение которой присваивается один раз во время выполнения (runtime) и может быть вычислено динамически. const — это строгая константа, значение которой известно на этапе компиляции (compile-time), вычисляться во время выполнения не может и часто используется для оптимизации UI-элементов.
Что означает String??
Это nullable-тип, который указывает, что переменная может хранить как строковое значение, так и null. В Dart включён режим Sound Null Safety, поэтому по умолчанию все переменные считаются non-null. Знак ? явно разрешает хранение пустого значения, перенося проверку на компилятор.
Чем Future отличается от обычного значения? Что означает await с точки зрения потока выполнения?
Обычное значение доступно сразу в момент вызова, тогда как Future<T> представляет собой объект-обещание, который вернёт результат в будущем (например, после сетевого запроса или таймера). Ключевое слово await приостанавливает выполнение текущей функции до получения результата Future, но не блокирует основной поток (event loop). Это позволяет Dart продолжать обрабатывать другие события, пока ожидание не завершится (в отличие от блокирующего Thread.Sleep).
Зачем в Dart именованные конструкторы, если в C# есть перегрузка?
В Dart отсутствует перегрузка конструкторов (constructor overloading). Чтобы предоставить несколько вариантов создания объекта с разными наборами параметров или логикой инициализации, используются именованные конструкторы (например, Todo() или Todo.fromJson()). Это делает код более читаемым, явным и позволяет чётко разделять сценарии создания экземпляров без конфликта сигнатур.
